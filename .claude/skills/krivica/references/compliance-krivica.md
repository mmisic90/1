# Compliance извештај — `krivica` v2

> Овај фајл описује детаљан формат mini compliance извештаја који `krivica` генерише после сваког кривичног акта. Извештај се додаје **ИСПОД** извештаја од `pravna-analiza`, не уместо њега.

## ЗАШТО ПОСТОЈИ ПОСЕБАН KRIVICA COMPLIANCE

- `pravna-analiza` извештај покрива **аналитичке фазе** (Ф0–Ф5).
- `krivica` извештај покрива **специфичне кораке писања кривичног акта** (К0–К11 из SKILL.md).
- Без оба, корисник не види комплетну слику квалитета.

## КРИТЕРИЈУМИ ЗА ФИНАЛНУ ОЦЕНУ

| Оцена | Услов |
|---|---|
| 95–100% | Сви кораци ✅, 0 П4, 0 П3 неоткривених, verifikator ✅ |
| 85–94% | Сви кораци ✅, 0 П4, 1–2 ставке у „за ручну проверу" |
| 70–84% | 1 корак ⏭️ са оправдањем, 0 П4 |
| 50–69% | 2 корака ⏭️, потенцијална празнина |
| < 50% | БЛОКИРАНО — не шаље се кориснику без корекције |

## АКЦИОНЕ ПОРУКЕ

### Ако К0 = ⛔ (нема handoff-а)
```
⚠️ RADIM U FALLBACK REŽIMU. Quality će biti niži nego sa 
pravna-analiza handoff-om. Ako imaš dokument za analizu, 
učitaj ga pa ću restartovati pun stack.
```

### Ако К3 = ⛔ (istrazivanje-prakse није активан)
```
⛔ STAJEM. Ne pišem krivični akt bez verifikovane prakse.
Aktiviraj istrazivanje-prakse pa nastavljam.
```

### Ако К8 показује П4 > 0
```
⛔ KRITIČNO. U dokumentu ima [N] tvrdnji klase P4 
(neproverene / potencijalne halucinacije). Dokument se 
NE ŠALJE korisniku dok se ne očisti. Vraćam se na:
- [lista P4 tvrdnji]
```

### Ако verifikator врати DIFF ❌
```
⚠️ verifikator je našao [N] neslaganja između mog 
dokumenta i izvora. Ispravljam:
- [lista neslaganja]
```

## ПРАВИЛА

1. Извештај је **обавезан** — нема краја без њега.
2. Извештај се приказује у чату **одмах испод** одговора са генерисаним документом.
3. Ако корисник каже „само ми дај документ без извештаја" → и даље се приказује извештај. Ово је дефанзивни механизам против халуцинација.
4. „За ручну проверу" мора имати **минимум 3** ставке.
5. Ако нема ниједне ставке за ручну проверу — то значи да није размишљано довољно, и цела операција се враћа на `verifikator`.
